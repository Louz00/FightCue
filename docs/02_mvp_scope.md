# MVP Scope

## Platform target

- Android launch target
- iOS-ready architecture from day one
- shared Flutter codebase for both platforms

## Sports in MVP

- Boxing first
- MMA second
- Kickboxing after the first reliable ingestion milestone

## Must-have MVP features

- upcoming events list
- event detail
- expandable event cards for fast fight-card viewing
- local timezone conversion
- event-level favorites
- fighter follows
- reminder presets
- watch availability by selected country
- calendar export
- anonymous user profile
- optional account creation and linking
- basic settings
- quiet ad support in free tier
- premium state framework

## Nice-to-have if time allows

- bout-level favorites
- premium filters
- generated fighter avatars
- richer source/broadcast metadata
- initial organization leaderboards where data is clearly source-backed

## Explicitly out of scope for v1

- live scoring
- exact bout-start guarantees
- social feed or community features
- complex account onboarding
- broad source coverage on day one
- noisy ad formats that interrupt primary flows

## Free vs premium

### Free

- browse upcoming events
- see local event time
- see watch information by country
- follow a limited number of events
- follow a limited number of fighters
- basic reminder presets
- basic calendar export
- quiet ads

### Premium

- no ads
- unlimited favorites
- unlimited alerts
- advanced reminder presets
- future premium filters
- future bout-level tracking

## Identity stance

- app works anonymously by default
- optional account exists for sync, restore, and future cross-device continuity
- premium purchases should not require an account, but optional account linking improves restore and portability

## Leaderboards

- leaderboards are desirable, but must be clearly labeled by source and organization
- for boxing this likely lands after MVP unless a defensible ranking source is chosen
